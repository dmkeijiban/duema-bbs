# 既存投稿本文の上書き禁止ルール

> 制定: 2026-05-25  
> 背景: 2026-05-02の物理削除事故を踏まえ、削除禁止ルールと同等の強さで「本文上書き禁止」を制定する

---

## 絶対ルール（例外なし）

### 1. 既存 posts.body を勝手に UPDATE しない
- スクリプト・自動化・バッチ・AI から `posts.update({ body })` を呼び出すことを禁止する
- 管理画面の `adminUpdatePost()` のみが唯一の合法的な呼び出し元であり、それも下記制約を満たす場合のみ

### 2. 既存コメント・投稿の本文を勝手に UPDATE しない
- ユーザーが投稿した本文はユーザーの財産・掲示板の運用履歴として保全する
- 内容が不適切でも「削除（is_deleted=true）」が正しい対処法。本文の書き換えではない

### 3. リバイバル・補完は新規 INSERT のみ。既存コメントの上書き禁止
- コメント0件スレへの復活コメントは `posts` テーブルへの新規 INSERT で行う
- 既存コメントを「置き換える」「書き直す」「補完する」操作は禁止
- INSERT するのは「post_count が 0 のスレ」のみ。既存コメントがあるスレへの自動 INSERT も禁止

### 4. 既存コメントはユーザー投稿・運用履歴として保全する
- 誰がいつ何を書いたか、その記録は消さない・改変しない・移動しない
- データは「消さない・上書きしない・残す」を基本方針とする

### 5. 例外的に編集が必要な場合（誤字修正等）の必須プロセス
管理者が手動で1件だけ修正する場合でも、以下を必ず実施すること：

1. **dry-run**: 変更対象の特定と変更内容の事前確認
2. **対象ID一覧**: 変更する `post.id` を列挙
3. **変更前本文**: 現在のbodyをそのまま提示
4. **変更後本文**: 変更後のbodyを提示
5. **差分**: 変更前後の差分を明示
6. **ユーザー確認**: 上記を提示してからユーザーの明示的な「yes」を待つ
7. **実行**: 確認後に1件ずつ `adminUpdatePost()` 経由で実行

### 6. ユーザー確認なしの本文 UPDATE は禁止
- 「たぶん大丈夫」で実行しない
- 確認プロンプトを省略しない
- 「まとめてやっておきました」は禁止

### 7. 一括 UPDATE は禁止
- 複数の posts.body を一度に書き換えることは禁止
- ループで回しても実質的な一括 UPDATE となるので禁止
- 変更できるのは1件ずつ、確認を挟みながら

### 8. 復旧・再生・リバイバルで既存コメントを置き換えるのは禁止
- 「消えたコメントを復元する」目的でも、既存のbodyを上書きしてはならない
- 失われたコメントは「失われた」まま記録に残し、新たな INSERT で補完する
- 補完 INSERT も上記のルール（0件スレのみ・ユーザー確認・dry-run）を満たすこと

### 9. 既存データは「消さない・上書きしない・残す」を基本方針にする
- 削除 → `is_deleted = true`（物理削除禁止）
- 非表示 → `is_archived = true`
- 本文変更 → 原則禁止。例外は管理者手動・1件・確認済みのみ
- 移動・変換 → 禁止（postsテーブル間の移動も禁止）

## コードレベルの制約

### `adminUpdatePost()` に追加された保護

`src/app/admin/actions.ts` の `adminUpdatePost()` は以下の保護が入っています：

1. **大きな警告コメントブロック**: 禁止用途を明示
2. **変更前本文の事前取得**: 実行前に既存 body を SELECT
3. **監査ログ出力**: 変更前後の本文を `console.warn` で記録（Vercel Functions ログに残る）
4. **単一IDガード**: `.eq('id', postId)` で1件のみを対象

### スクリプトからの呼び出し禁止

`generate-empty-thread-revival.mjs` および `insert-approved-revival-comments.mjs` は：
- 既存コメントが存在するスレへの INSERT を行わない（`checkExistingPosts` でガード）
- `posts.update({ body })` を一切呼び出さない

---

## 違反パターンと対処

| 違反パターン | 対処 |
|---|---|
| スクリプトが `posts.update({ body })` を呼び出す | スクリプトを停止し、変更を revert する |
| AI が既存コメントを「書き直す」提案をする | 提案を却下し、新規 INSERT の方針に戻す |
| 一括 UPDATE で本文を上書きしようとする | 即時停止・ユーザーへ報告・dry-run から再設計 |
| 確認なしで本文 UPDATE を実行する | `adminUpdatePost` の監査ログで事後確認・再発防止を記録 |
