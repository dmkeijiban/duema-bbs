# X週次PDCAルール & ジャンル別テンプレート

> 目的：週次でClaudeが下書きを生成→Typefully予約する流れを、**手直しゼロに近づける**ための手順とテンプレ集。
> 文体ルールの詳細は [`x-posting-style-guide.md`](./x-posting-style-guide.md) を参照。
>
> ⚠️ **絶対ルール**：既存の投稿済みポスト・Typefully予約投稿・DBデータは**削除・上書き・修正しない**。新規の `typefully_create_draft` 追加のみ。重複スロットはスキップ。

---

## 1. スロット別の役割（毎日4枠）

| 時間 | 狙う反応 | 主役ジャンル | 口調・形式 |
|---|---|---|---|
| **7:00** | いいね・共感 | 優勝🏆（軽い共感）・老人会 | 短く軽い。朝にサッと参加できる |
| **12:00** | リプ・会話量 | 優勝🏆主砲・VS型（週1〜2）・選手権 | 最大インプ狙い。世代差・微妙ライン |
| **19:00** | リポート・拡散 | 物議・高騰・「説」・**チャレンジ枠（水曜）** | 2〜4行、砕け語、ハッシュタグ無し |
| **22:00** | プロフクリック | 選手権・懐古・黒歴史 | URL・掲示板誘導禁止。夜の語り |
| 結果発表（不定期） | 保存・再訪 | 入賞/金賞/最優秀賞の発表 | 一覧型。前回お題の回収 |

> 1週間分を作るとき、**4枠すべてが「リプ狙い」型に偏っていないか**必ずチェックする。

## 2. ジャンル別テンプレート

各テンプレは「骨格」。**そのまま出さず、必ず [`x-posting-style-guide.md`](./x-posting-style-guide.md) の文体ルール（口調・自己開示・改行）を被せてから予約する。**

### 2-1. 7:00 優勝🏆（軽い共感）
```
デュエマで
「○○」
を言えた人が優勝🏆
```
- お題は具体・懐古寄り（例：「あのころ一番強かった友達のデッキ」）。
- 末尾に**自己開示の一言**を足すと強い（例：`僕はアッシュ！…ｗ`）。
- 抽象お題（「好きなカード」）は禁止 → 必ず一捻り。

### 2-2. 12:00 VS型（週1〜2・[昇格]済み正規テンプレ）
```
[カードA] vs [カードB]

どっちが好き？？
```
- 末尾は「どっちが好き？」固定。`コメントで教えてください👇` に変えない。
- 軸をローテーション：カード対決 → デッキ対決 → 時代対決 → 文明対決 → 呼び方世代差（被り防止）。
- 同キャラ進化前後・ライバルカード対決は話題性高め。

### 2-3. 12:00 優勝🏆（昼の主砲）
- 7:00と同型だが、**最大インプ狙い**なので最も尖ったお題を昼に置く。
- 「ズレ・微妙ライン・世代差」を必ず含める。

### 2-4. 19:00 物議・「説」（拡散狙い）
```
デュエマって
○○多くない？ｗ
```
または
```
○○なのは
○○だから説
```
- 2〜4行、砕け語、ハッシュタグ無し。
- 末尾は「？」「説」「わかる人🙋」で締める。
- 賛否が割れる/世代で意見が分かれるお題を選ぶ。

### 2-5. 19:00 チャレンジ枠（水曜のみ・[未テスト]アイデア）
- `X-ideas.md` の `[未テスト]` から**週1本だけ**選ぶ。
- 19時枠フォーマット（短文2〜4行・砕け語・締めは「？」「説」「🙋」）を厳守。
- 清書必須：アイデアの素のままは使わず、フォーマットに合わせてClaudeが整える。
- 登録したら `X-ideas.md` を `[未テスト]→[テスト中]` に更新し、テスト投稿日を記入。
- `[未テスト]` がゼロの週は**スキップ**して通常テンプレで代替（強制しない）。

### 2-6. 22:00 選手権（懐古・プロフクリック狙い）
```
デュエマの
「○○」
選手権を開催します。

カード名だけでもOKです。
```
- URL・掲示板誘導は**本文に絶対入れない**（過去に7本全部誘導になり全修正された）。
- 末尾CTAは「リプ」表記（`リプで教えてください👇` / `気軽にリプ下さい！`）。
- 「小学生の頃使ってたカード選手権」が最高記録（473Kインプ/1,700いいね）→ **懐古×自分ごと化**が鉄板。

### 2-7. 22:00 老人会（懐古・自己開示）
```
デュエマ老人会

「これ絶対誰も知らんだろ」
って思うカードを教えてください👇
```
- 「老人会」の一語で文脈を立てる。前置き不要。
- 引退勢が「わかる」と言いやすいマイナー寄りのお題。

### 2-8. 結果発表（入賞/金賞/最優秀賞）※新規ジャンル
前回のお題（選手権・優勝🏆）を**回収**して発表する一覧型。継続再訪を生む。
```
先日の「○○選手権」結果発表🏆

🥇最優秀賞：[カード名]
🥈金賞：[カード名]
🥉銅賞：[カード名]

みんなありがとうｗ また開催します
```
- 元お題の投稿と**必ず対応**させる（宙に浮いた発表にしない）。
- 受賞理由を一言添えると温度感が出る。
- リスク：クオリティが落ちると離脱 → TOP3程度で短く完結。

## 3. フォーマット多様性キャップ（被り防止）

同じ週に同型を入れすぎると週間インプが下がる。以下を上限とする。

| 型 | 週の上限 | 間隔 |
|---|---|---|
| 優勝🏆 | 3本 | 2日空ける |
| 選手権 | 3本 | 2日空ける |
| VS型 | 2本 | 3日空ける |
| 老人会 | 2本 | 3日空ける |
| 物議・「説」 | 上限なし | — |
| 同一テーマ系統（例：シールドトリガー系） | **週1本** | — |

- 7:00 / 12:00 / 22:00 は常に通常テンプレ。チャレンジ枠はここに侵食しない。
- 週前半（月〜火）は通常テンプレで固め、水曜にチャレンジする。

## 3.5. 重複防止ルール（テーマ・問い・切り口の被り検知）

> §3 は「型」の被り、ここは「中身（テーマ・問い・切り口）」の被りを防ぐ。**新規生成の前に必ず通す。**

### 基本ルール
- 原則、**一度使った投稿テーマ・問い・切り口は再使用しない**。
- 似た内容を使う場合でも、**最低2〜3日は空ける**。
- **同じカード名・同じ問い方・同じ構文が連続しない**ようにする。
- 同じテーマを再利用するときは、**必ずニュアンスを変える**。
  - ✅ OK：「好きなカード」→「今でも売れないカード」→「子どもの頃の切り札」
  - ❌ NG：「一番好きなカード」→翌日「好きなカード教えて」
- **優勝🏆・選手権・老人会・物議・違和感**など、同じ型でも**中身の問いが近すぎる**ものは避ける。

### 生成前の必須チェック（毎回）
1. **新規生成の前に、直近7日分の公開済み投稿・予約投稿を確認する**（§付録のデータソース早見表でTypefullyから取得）。
2. **直近2〜3日以内に近いテーマがある場合は、別テーマに差し替える**。
3. どうしても近いテーマを使う場合は、**対象・感情・時間軸・答え方のうち最低1つを変える**。

| ずらす軸 | 例 |
|---|---|
| 対象 | カード → デッキ → 環境 → プレイヤー像 |
| 感情 | 好き → 後悔 → 懐かしい → 悔しい |
| 時間軸 | 今 → あのころ → 引退時 → 初めて買った日 |
| 答え方 | カード名1個 → VS二択 → 🙋挙手 → 数字・金額 |

> 文体ガイド側の同ルールは [`x-posting-style-guide.md`](./x-posting-style-guide.md) §6 を参照。

## 4. 週次PDCA手順（Claudeが毎週実行）

### ステップ0：空き週の特定（最初に必ず）
1. `typefully_list_drafts(social_set_id=306286, status="scheduled")` で予約済みを取得。
2. 最後の予約日付の**翌週月曜**を登録対象週の開始日とする。
3. 「○/○週（月〜日の7日間）に登録します」と一言添えてから進む。
4. **登録済み期間の既存予約には一切触らない**。空きスロットだけに追加。

### ステップ1：完成形に文体を寄せる（最重要）
1. `typefully_list_drafts(social_set_id=306286, status="published", order_by="-published_at")` で直近の**公開済みポスト＝手直し後の完成形**を確認。
2. 公開済みの口調・改行・CTAの温度感に次週案を寄せる。**古い固定テンプレをそのまま回さない。**
3. `X運用.md` の「📝 ユーザー修正ログ」を読み、前回直されたパターンを反映。

### ステップ2：生成 → 自己チェック → 予約
1. **生成前に直近7日の公開済み・予約投稿を取得**し、重複防止ルール（§3.5）でテーマ・問い・切り口の被りを確認。被るものは別テーマに差し替える。
2. ジャンル別テンプレ（§2）で骨格を作る。
3. **生成直後に [`x-posting-style-guide.md`](./x-posting-style-guide.md) §5 の自己チェックリストを全項目通す**（重複防止項目を含む）。NOがあれば直す。
4. 水曜19時にチャレンジ枠（§2-5）を1本割り当て、`X-ideas.md` を更新。
5. 残り枠を通常テンプレで埋める。多様性キャップ（§3）と重複防止（§3.5）を確認。
6. `typefully_create_draft` で**新規追加のみ**。重複スロットはスキップ。

### ステップ3：結果の記録
1. 週次X結果を受け取ったら、チャレンジ枠を過去4週の同枠baselineと比較。
2. 昇格（1.5倍以上）/ 継続 / 没（0.7倍以下かつリポスト0）を判定し `X-ideas.md` 更新。
3. 昇格は該当スロットの正規テンプレに追加。`X運用.md` のPDCAログに記録。

## 5. 手直しゼロに近づけるための運用チェックリスト（週次・予約前）

下書き一式を予約する前に、**週単位**で以下を確認する（各下書き単位のチェックは style-guide §5）。

- [ ] 直近の公開済みポストを見て、口調・改行を**今週の完成形**に寄せたか
- [ ] 「📝 ユーザー修正ログ」の前回修正を反映したか
- [ ] 4枠が「リプ狙い」型だけに偏っていないか（いいね/拡散/プロフクリックも配分）
- [ ] フォーマット多様性キャップ（§3）を超えていないか
- [ ] 同週内でテーマ系統が被っていないか（同系統 週1本）
- [ ] **【必須ゲート】直近7日分の公開済み投稿・予約投稿を確認してから新規生成したか**（§3.5）
- [ ] 直近2〜3日以内に近いテーマが無いか（あれば別テーマに差し替えたか）
- [ ] 同じカード名・同じ問い方・同じ構文が連続していないか
- [ ] 同テーマ再利用時に対象・感情・時間軸・答え方のどれかを変えたか
- [ ] 水曜19時にチャレンジ枠を入れ、`X-ideas.md` を `[テスト中]` に更新したか
- [ ] お題系すべてに自己開示の一言を入れたか
- [ ] 全投稿でURL・掲示板誘導が本文に無いか
- [ ] 全CTAが「リプ」表記か
- [ ] 既存予約・既存投稿に一切触っていないか（新規追加のみ）
