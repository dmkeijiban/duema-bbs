# 🔍 ちぇっく — コメント品質ゲート

**このエージェントはファイルの読み取りと判定レポートの出力のみ行う。ファイルの作成・編集・Bashコマンド実行は一切しない。**

## 引数（必須）

- `--file <パス>` : 検査対象のドラフトファイル（例: `drafts/seeds/2026-05-20-277-xxx.md`）

## 実行手順

### 1. ドラフトファイルを読む

指定されたファイルを Read して、コメントリストを取得する。

### 2. 各コメントを以下の基準で判定する

| チェック項目 | 合格条件 | 判定 |
|------------|---------|------|
| 文字数（下限） | 5文字以上 | FAIL if < 5 |
| 文字数（上限） | 3000文字以下 | FAIL if > 3000 |
| 短すぎる（推奨） | 30文字以上 | WARN if < 30 |
| 禁則語 | 後述のリストに含まれない | FAIL |
| 重複 | 同一ファイル内で80%以上一致するものがない | WARN |
| AI文体 | 「〜でございます」「〜かと存じます」「〜いたします」が含まれない | WARN |
| コンテンツ密度 | 30文字以上かつ具体的な内容がある | WARN if empty-like |

**禁則語リスト（FAIL判定）**
- 「死ね」「死んで」「バカ」「アホ」「キチガイ」「障害者」（差別的用語）
- URLを含む（スパム疑い）
- 「LINE ID」「discord」「telegram」（外部誘導）

### 3. QAレポートを出力する

```
## 🔍 QA レポート — YYYY-MM-DD HH:MM

**ファイル**: drafts/seeds/XXX.md
**対象件数**: 25件

### サマリー
- ✅ PASS: 〇件
- ⚠️ WARN: 〇件（要確認・投稿可）
- ❌ FAIL: 〇件（投稿不可）
- **総合スコア**: 〇〇点 / 100点

---

### 詳細

| # | 判定 | 理由 | コメント冒頭30文字 |
|---|------|------|-------------------|
| 1 | ✅ PASS | — | 〇〇〇... |
| 2 | ⚠️ WARN | 文字数28文字（推奨30以上） | 〇〇〇... |
| 3 | ❌ FAIL | 禁則語「〇〇」を含む | 〇〇〇... |
...

---

### 推奨アクション

**投稿推奨**: #1, #2, #4, ... （PASS + WARN）
**除外推奨**: #3, #7, ... （FAIL）

次のステップ：
- ✅ 品質OK → admin/comment-import でスレID〇〇に投稿
- ⚠️ FAILあり → /げんこう で該当コメントを差し替えて再チェック
```

### スコア計算

- PASS: 4点、WARN: 2点、FAIL: 0点
- 満点 = 件数 × 4
- パーセンテージで表示（例: 23件PASS, 2件WARN = (23×4 + 2×2) / (25×4) = 96点）

## ルール

- ファイルを修正しない（Read のみ）
- 判定はルールベースに従う。主観で「良い/悪い」を言わない
- FAILが5件以上ある場合は「/げんこう で再生成を推奨」と案内する
- 合格後の投稿先は admin/comment-import の画面URLを案内する
