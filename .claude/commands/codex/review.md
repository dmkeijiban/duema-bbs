# ⚡ Codex コードレビュー

引数を解析して PR またはファイルをレビューする。

## 使い方

```
/codex:review pr=<番号>
/codex:review file=<パス>
/codex:review          # 引数なし → 直近のステージング変更をレビュー
```

## 実行手順

### PR レビュー（`pr=<番号>` 指定時）
1. `gh pr view <番号>` で PR の概要を確認
2. `gh pr diff <番号>` で差分を取得
3. 以下の観点でレビュー：
   - **正確性**：ロジックのバグ、型の不整合、null/undefined の見落とし
   - **Next.js 規約**：`node_modules/next/dist/docs/` の最新規約に従っているか
   - **Supabase**：RLS 設定漏れ、N+1クエリ、型安全性
   - **セキュリティ**：API キー露出、SQL インジェクション、XSS
   - **パフォーマンス**：不要な再レンダリング、大きな画像、遅延ロード未使用
   - **保守性**：重複コード、命名の一貫性、コメント不足
4. 問題点を **Critical / Warning / Suggestion** の3段階で分類して報告
5. 修正が必要な Critical 項目があれば、修正コードを提示する

### ファイルレビュー（`file=<パス>` 指定時）
1. 指定ファイルを読む
2. 上記と同じ観点でレビュー
3. 改善案のコードスニペットを添付する

### 引数なし
1. `git diff --staged` で現在のステージング変更を確認
2. 変更ファイルを列挙してレビュー

## ルール

- Critical 項目がある場合：修正せずに報告し、ユーザーの判断を仰ぐ
- `lint/build` を通していないコードには必ずその旨を明記する
- AGENTS.md の「触らないファイル」指定があれば尊重する
