# ✍️ ライターロール

引数を確認して対応するモードで動く。引数がない場合は使い方を表示する。

---

## 📰 記事生成モード — `--type article --url <URL>`

DMWiki の商品ページ URL を受け取って記事を自動生成・インポートする。

1. `npm.cmd run content:article -- --url <URL>` を実行
2. 生成された `drafts/articles/*.md` の品質チェック JSON を確認
3. スコアが低い（70点未満）場合は問題点を報告して再生成を提案
4. スコアOKなら `npm.cmd run content:import-draft -- --file <draft.md>` でインポート
5. 管理画面URL（`/admin/pages`）と記事スラッグを報告
6. 公開するかどうかを確認する（自動公開はしない）

**ルール：**
- 記事は必ず `published: false` の下書き状態でインポート
- AI量産感を避けるため：具体的なカード名・コスト・能力・歴史的文脈を含める
- 画像は主役カードのみ公式カード検索から取得（全カードへの拡張は別タスク）

## 🐦 X投稿下書きモード — `--type x --topic <テーマ>`

テーマを受け取ってX投稿文を生成し、Typefully に下書き登録する。

1. テーマから最適な投稿タイプを選択：
   - `winner` 優勝🏆（7:00枠）
   - `odd` 違和感/物議（12:00 or 19:00枠）
   - `retro` 老人会/黒歴史（12:00枠）
   - `silhouette` シルエットクイズ（19:00枠）
   - `bbs` 掲示板共有（22:00枠）
   - `kouton` 高騰下落（22:00枠、市場動向がある時のみ）
2. 140文字以内の投稿文を3パターン生成
3. ユーザーが選んだパターンを Typefully MCP で下書き登録（スケジュールなし）
4. Typefully 下書きURLと投稿内容を報告

**ルール：**
- Typefully への登録は下書きのみ。スケジュール設定・投稿は人間が行う
- 優勝投稿は1日1件まで（既存 x_posts を確認して重複回避）
- キリン構文（長文解説）は人間が作成。AIは生成しない

---

## 🔍 SEO最適化モード — `--type seo --page <スラッグ>`

指定ページのメタデータを改善する。

1. `src/app/` 内の対象ページのメタデータを確認
2. 現在のtitle・descriptionを表示
3. 改善案を3パターン提示（キーワード・CTR・文字数を考慮）
4. 承認後にファイルを更新

---

## 使い方（引数なしの場合）

```
/writer --type article --url https://dmwiki.net/DM26-RP1
/writer --type x --topic "今日の優勝デッキ"
/writer --type seo --page article-dm26-rp1
```
